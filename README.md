# 🏦 IndusCredit Risk Decisioning Platform
###  DataSprint — Loan Default Risk Assessment

An end-to-end ML-powered credit risk system that predicts loan default probability using LightGBM, explains decisions with SHAP, and serves results through a FastAPI backend + Streamlit dashboard.

---

## 📌 Problem Statement

IndusCredit Finance (a mid-size NBFC) has a portfolio of ₹12,800 crore with an NPA ratio of **4.8%** — above the RBI comfort threshold of 3.5%. Their legacy scorecard uses only 3 features (credit score, income, loan amount).

**Goal:** Build an ML model that:
- Compares 3 models (Logistic Regression, XGBoost, LightGBM)
- Provides **SHAP-based explanations** for every decision
- Serves results via a real-time credit decisioning dashboard

## 🚀 How to Run Locally

### Step 1 — Clone the repository

```bash
git clone https://github.com/YOUR_USERNAME/datasprint.git
cd datasprint
```

### Step 2 — Install dependencies

```bash
python -m pip install fastapi uvicorn streamlit pandas numpy lightgbm shap scikit-learn python-dotenv openai google-cloud-vision requests plotly Pillow python-multipart category_encoders xgboost optuna
```

### Step 3 — Set up environment variables

Create a `.env` file in the root folder:

```
OPENAI_API_KEY=your_openai_key_here
HF_API_TOKEN=your_huggingface_token_here
```

### Step 4 — Add your data and models

Place the following files before running:
- `data/loan_train.csv`
- `data/loan_test.csv`
- All `.pkl` files inside `models/` (generated from the Colab notebook)

### Step 5 — Run the backend (Terminal 1)

```bash
cd datasprint/backend
python -m uvicorn main:app --reload --port 8000
```

API docs available at: `http://localhost:8000/docs`

### Step 6 — Run the frontend (Terminal 2)

```bash
cd datasprint/frontend
python -m streamlit run app.py
```

Open in browser: `http://localhost:8501`

## 📓 Generating the Model (Google Colab)

The full ML pipeline lives in `loan_default_model.ipynb`. Open it in [Google Colab](https://colab.research.google.com), upload `loan_train.csv` and `loan_test.csv`, and run all cells in order. At the end, add a cell to save all artifacts:

```python
import pickle

with open('lightgbm_model.pkl',     'wb') as f: pickle.dump(best_lgb, f)
with open('target_encoder.pkl',     'wb') as f: pickle.dump(te, f)
with open('feature_columns.pkl',    'wb') as f: pickle.dump(list(X.columns), f)
with open('outlier_bounds.pkl',     'wb') as f: pickle.dump(outlier_bounds, f)
with open('best_threshold.pkl',     'wb') as f: pickle.dump(BEST_THRESHOLD, f)
with open('ohe_cols.pkl',           'wb') as f: pickle.dump(OHE_COLS, f)
with open('target_encode_cols.pkl', 'wb') as f: pickle.dump(TARGET_ENCODE_COLS, f)
```

Download the `.pkl` files from Colab and place them in your `models/` folder.

## 🔍 Feature Engineering

8 new features created on top of the raw dataset:

| Feature | Formula |
|---|---|
| `loan_to_income_ratio` | `loan_amount / annual_income` |
| `dti_credit_risk` | `dti_ratio / (credit_score / 700)` |
| `income_per_year_employed` | `annual_income / (employment_years + 1)` |
| `emi_estimate` | Calculated from loan amount, rate, tenure |
| `emi_to_income_ratio` | `emi_estimate / (monthly_income)` |
| `savings_to_loan_ratio` | `savings_balance / loan_amount` |
| `payment_stress_score` | `missed_payments × bureau_enquiries` |
| `credit_utilization` | `num_existing_loans × dti_ratio` |
